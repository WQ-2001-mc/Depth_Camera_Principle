# 存内计算用于双目匹配代价体构建：论文总结与方案构思

> 整理范围：本目录内 3 份 PDF  
> 整理日期：2026-07-16  
> 核心问题：存内计算如何缓解双目匹配代价计算中的数据搬运、计算吞吐和阵列利用率瓶颈，以及这些思想如何转化为可落地的 FPGA/SGBM 研究方案。

## 1. 先给结论

这组论文形成了一条清晰的技术演进路线：

1. **MC-CIM（DAC 2022）**首先证明，可在 SRAM 阵列附近加入轻量 XOR 计算单元，通过可重构数据映射支持 Census/Hamming、SAD 等不同匹配代价，减少处理单元与存储器之间的数据搬运。
2. **CV-CIM（ISSCC 2023）**把研究对象从单一“匹配代价计算”提升到更完整的“代价体构建”，进一步解决三类问题：功能受限、模拟计算裕量下降、刚性阵列利用率低。
3. **CV-CIM（JSSC 2025）**是 ISSCC 2023 工作的完整期刊扩展版，而不是第三条独立路线。它补充了算法精度、芯片测量、PVT 鲁棒性、不同 L0/L1/L2 代价函数的性能和完整消融结果。

对当前双目深度相机与 FPGA/SGBM 研究，最值得吸收的并不是“直接在 FPGA 上复刻模拟 SRAM-CIM”，而是以下四个可迁移思想：

- 以 **XOR 为统一基础算子**，重构 Census、SAD、相关性等多种代价函数；
- 利用相邻像素的相似性做 **差分编码、稀疏化和零值跳过**；
- 围绕滑动窗口和视差搜索设计 **行列双维度的数据复用与计算/加载重叠**；
- 根据纹理、遮挡、反光和主动散斑质量 **动态选择代价函数或精度档位**。

需要严格区分两件事：

- 论文中的 SRAM-CIM 是定制 28 nm ASIC 宏，涉及 6T SRAM、模拟电流累加、ADC 和 PVT 校准；
- FPGA 中的 BRAM/URAM 不能直接实现同样的晶体管级存内运算。FPGA 阶段应称为“存储近计算”“面向存储的数据流优化”或“CIM-inspired architecture”，除非后续确实开展定制芯片设计。

## 3. 问题背景：为什么双目匹配适合研究存内计算

### 3.1 代价体构建的计算形式

对校正后的双目图像，通常需要为左图像素 `(x, y)` 和右图候选像素 `(x-d, y)` 计算匹配代价：

```text
C(x, y, d) = D(F_L(x, y), F_R(x-d, y))
```

其中 `d` 为候选视差，`D` 可以是：

- L0/Hamming 距离：典型代表为 Census 描述子异或后的 popcount；
- L1/Manhattan 距离：典型代表为 AD/SAD；
- L2/相关性距离：可用于特征相关或 CNN 特征代价体。

随着图像宽高、帧率、窗口大小和视差范围增大，相同或相邻数据会被反复读取。MC-CIM 给出的应用量级是 1280×720、50 FPS 时约 700 Mb/s 原始图像流、约 1 Tb/s 存储带宽需求和超过 470 GOP/s 的计算需求；CV-CIM 则以实时 30 FPS 场景说明代价体构建会产生超过 0.25 Tb/s 的持续带宽压力。[1, pp. 1-2; 3, pp. 1-3]

### 3.2 真正的瓶颈不只是算术运算

这组论文反复强调，代价计算的核心瓶颈是“算术 + 数据组织 + 数据搬运”的耦合，而不只是 XOR 或减法本身。JSSC 论文引用的数字加速器案例中，存储访问功耗为 31.2 mW，代价计算本身为 10.2 mW，即数据访问功耗约为计算功耗的 3.06 倍。[3, p. 1]

在传统 FPGA/ASIC 流水线里，即使增加并行处理单元，也仍可能遇到：

- 右图候选窗口和左图参考窗口反复读取；
- 不同视差导致数据对齐和地址生成复杂；
- Census、SAD、相关性等代价函数的数据宽度和归约方式不同；
- 视差搜索窗口滑动时，阵列或 BRAM 端口利用率不均；
- 代价体生成后还要送入 SGM/SGBM 路径聚合，形成新的带宽压力。

## 4. 论文一：MC-CIM（DAC 2022）

### 4.1 研究目标

MC-CIM 试图解决三个问题：

1. 传统 SRAM-CIM 多面向 MAC，而双目代价需要 XOR 累加、绝对差、比较和加法等多种运算；
2. 固定计算电路难以适配不同场景下的 Census 或 SAD；
3. 不同算法和位宽的数据复用模式不同，需要可配置的数据映射和数字归约结构。[1, pp. 1-2]

### 4.2 架构和关键机制

MC-CIM 由 64 个 compute array 组成。每个阵列包含 16×64 个 6T SRAM 单元，每列配置一个轻量计算单元；8 个阵列组成 compute tile，并通过分层移位/加法树（HSAT）合并部分和。[1, pp. 2-3]

核心设计包括：

- **对称 XOR 计算单元**：每列增加 5 个晶体管形成对称电流路径，降低不同输入组合造成的感知裕量重叠；晶体管开销约为每列 4.95%。
- **模拟归约**：最多 16 个逐位结果以电流形式横向累加，再由约 4 bit Flash ADC 转换为部分和。
- **HSAT**：支持对不同 bit-plane 的部分和进行移位、累加或旁路，可适配 2/4/8 bit 精度。
- **可重构运算**：以 XOR 为基础，通过输入固定、取反和数字后处理支持 XAC、MAC、ADD、SAC 和 SAD。
- **可配置映射**：针对 Census 和 SAD 的不同复用方向安排参考像素、候选像素、bit-plane 和阵列地址。

### 4.3 对 Census 和 SAD 的映射

对于 Census：

- 左图 Census bit-string 作为共享输入；
- 多个右图候选描述子存于不同 compute array/row；
- 阵列内执行 XOR 和 popcount；
- 简单 2:1 行地址 MUX 支持滑动视差窗口，避免给每个阵列配置复杂的独立地址网络。

对于 SAD：

- 多 bit 像素拆成 bit-plane 存入不同阵列；
- 比较结果控制绝对差的处理方向；
- HSAT 按位权合并部分和；
- 论文还利用不重叠列比较和 FIFO，声称可消除约 85% 的冗余比较计算。[1, pp. 4-5]

### 4.4 主要结果

MC-CIM 在 TSMC 28 nm 工艺下以 64 Kb 阵列进行评估，使用 KITTI 数据估计功耗和延迟：

- 平均能效约 277 TOPS/W；
- 对所选工作负载，比较表给出的延迟为 12.12 ms；
- Census/SAD 的复杂配置中，文中报告 SAD 延迟可超过 14.9 ms，最复杂 Census 配置约 18.7 ms；
- 当供电电压不低于 0.7 V 时，XOR 累加输出的线性度较稳定；
- Flash ADC 约占宏功耗的三分之一，局部计算单元接近一半。[1, pp. 5-6]

### 4.5 优点、局限和可借鉴点

优点：

- 首次把 SRAM-CIM 的研究对象明确对准双目匹配代价；
- 不是只加速固定 Census，而是考虑算法和位宽可重构；
- 数据映射、地址生成、阵列利用率与计算电路协同设计。

局限：

- 主要处理局部匹配代价，尚未系统支持 L0/L1/L2 统一代价体；
- 仍依赖 ADC，且需要离线调电压梯度；
- 某些配置下利用率仍低，论文也承认生产率不会随阵列规模线性增加；
- 评估聚焦代价计算，不代表完整 SGBM 或完整深度相机端到端性能。

对 FPGA 的直接启发是：将 Census 和 SAD 都拆解为“共享输入 + 多候选并行 + bit-plane 归约”，通过可编程移位加法网络复用同一套硬件，而不是为每种代价函数各做一套完全独立的流水线。

## 5. 论文二和三：CV-CIM（ISSCC 2023 / JSSC 2025）

### 5.1 从 MC-CIM 到 CV-CIM：问题升级为 3R

CV-CIM 将困难总结为三个 3R 问题：[3, pp. 1-3]

1. **Restricted logic**：传统 CIM 主要支持 AND/MAC，难以覆盖 Hamming、绝对差、加法、比较和相关性；
2. **Reduced computing margin**：模拟电流/电压的累加值越大，相邻量化等级间距越小，代价体又对逐像素误差敏感；
3. **Rigid array**：固定行列激活方式不适配滑动搜索窗口，导致多周期访问、低利用率和数据加载停顿。

### 5.2 总体架构

CV-CIM 包含 4 个 hybrid XOR-CIM group、顶层控制器、数字合并单元 DMU、临时缓冲、Canary-style BIST 和外围电路。每个 group 含 1 个 Pixel CIM 和 8 个 Vector CIM：[3, pp. 4-5]

- **Pixel CIM**：使用数字 XOR 和面积优化全加器，完成逐元素的比较、加减等处理；
- **Vector CIM**：使用电流域 XOR 做向量归约，并由相似性感知 ADC 量化；
- **DMU**：根据算法模式和 bit significance 做移位、累加和结果合并；
- 每个阵列为 16×64 cell，最多同时激活 32 个 XOR 计算单元；
- 4 bit Flash ADC 负责模拟到数字转换。

这种混合域设计的重要思想是：**不强求所有逻辑都在模拟存储阵列中完成**。逐像素、控制敏感的运算放在数字域；大规模向量归约放在模拟 CIM；再由数字单元完成精确合并。

### 5.3 XOR-derived logic：统一 L0/L1/L2 距离

CV-CIM 使用多位二进制表示（MBR），将每一 bit 表示为 `+1/-1` 并保留位权。借助 XOR 的可变换性质，组合出取反、加法、比较、乘法和归约，从而支持：

- L0：Census 描述子的 Hamming/XOR accumulation；
- L1：SAD 所需的比较、减法、绝对值和求和；
- L2：利用归一化向量欧氏距离与点积/相关性的关系实现。

论文估计，同等距离功能下，混合 Pixel/Vector CIM 相对独立数字距离单元可提高约 2.3 倍面积密度。[3, pp. 5-7]

### 5.4 相似性感知稀疏变换

普通 XOR 在输入 0/1 概率接近时容易产生较大累加值，压缩模拟感知裕量。CV-CIM 利用自然图像相邻像素连续性，将一个块内像素与参考像素比较，把原始 XOR 累加改写为对差分/相似性结果的稀疏 AND 型计算。

JSSC 完整实验给出的结果包括：[3, p. 11]

- block size 为 9 时，0-bit 稀疏率相对原始数据增加约 17%；block size 为 3 时增加约 36%；
- 稀疏率从约 55.2% 提高到 75.7%；
- 平均累加输出从 12.9 降到 5.7（block size 9），或降到 3.6（block size 3）；
- 输出整体降低约 4.13 倍；
- 平均归一化计算裕量由 84.1% 提高到 98.7%；
- all-zero bypass 额外降低约 5.2% 延迟。

这一机制对 FPGA 仍有价值，但价值不再体现为模拟“感知裕量”，而是体现为：

- 降低翻转活动和动态功耗；
- 提高零值跳过机会；
- 减少需要写入/传输的有效 bit；
- 为变长编码、稀疏压缩或分块早停提供条件。

### 5.5 行列双自由度外围

CV-CIM 从两个正交方向提高阵列利用率：[2, pp. 1-2; 3, pp. 8-11]

- **行方向**：epipolar-first 搜索优先计算极线上的关键候选，根据 pre-winner 和阈值跳过不显著候选；同时让不同阵列可在两个行地址之间灵活选择，以适配滑动区域。
- **列方向**：一半列计算时，另一半列加载下一批数据，通过奇偶列交替实现计算与加载重叠。

JSSC 给出的消融结果是：

- 单独行侧跳过对 L1/L2 较保守，延迟改善约 4.1%；
- 灵活行激活对 L0 可降低约 35.6% 延迟；
- 列侧计算/更新重叠分别使 L0、L1、L2 平均降低约 20.1%、20.7%、26.1% 延迟；
- 行列机制合并后，L0/L1/L2 的总延迟降低约 21%-47%；
- ISSCC 短文给出的保守跳过点是操作量减少约 11%，精度损失小于 2.3%。

### 5.6 芯片和算法结果

CV-CIM 使用 TSMC 28 nm CMOS 制造，芯片尺寸为 `0.622 mm × 0.622 mm`，即约 `0.387 mm²`，包含 36 Kb CIM group。工作范围为 0.6-0.9 V、50-286 MHz。[2, pp. 1-3; 3, pp. 9-13]

| 距离/算法 | 论文中的代表配置 | 延迟 | 能效 | 备注 |
|---|---:|---:|---:|---|
| L0 / Census | 64/128 disparity，9×9 或 11×11 | 4.49-12.00 ms | 约 947-1158 TOPS/W | 利用率 79.0%-94.5% |
| L1 / SAD | 64/128 disparity，7×7 或 9×9 | 4.72-14.71 ms | 约 453-505 TOPS/W | 利用率 43.7%-56.25%，受限于数据复用和并行度 |
| L2 / MC-CNN | concatenation/correlation | 17.4-31.5 ms | 约 989-1092 TOPS/W | 精度较高但计算最重 |

其他关键结果：

- 最佳峰值能效约 1158 TOPS/W，出现在 0.6 V 的 L0 场景；
- 平均峰值能效约 918.6 TOPS/W（0.6 V），0.9 V 时平均约 578.5 TOPS/W；
- 与所选近期 XOR/XNOR CIM 比较，能效提高约 1.05-5.29 倍，面积效率提高约 1.35-5.01 倍；
- 8 颗芯片在 0.9 V 下测试时平均线性拟合 `R² > 0.990`，最差芯片约 0.983；
- 论文表 III 中 L0 吞吐写为约 1259.24 GOP/s，但正文写成 1259.2 TOPs，存在单位不一致。做后续引用时应以表格口径为准，并注明操作归一化定义。

### 5.7 这套工作的关键局限

1. **只覆盖代价体构建**：没有完成完整 SGM/SGBM 的多方向路径代价递推、P1/P2 平滑约束、WTA、左右一致性检查和亚像素拟合。
2. **TOPS/W 不是端到端指标**：论文将距离、加法等归一化为基本 operation，不能与 FPGA 系统功耗或完整深度帧率直接等价比较。
3. **算法精度数字需谨慎解释**：表 III 的 Accuracy 是匹配距离阶段的特定 KITTI2012 设置，并非统一的 KITTI D1-all、EPE 或完整深度系统指标。
4. **模拟 CIM 有专属成本**：ADC、PVT 漂移、非线性、BIST 和参考阈值都需要面积和功耗；0.6 V 的峰值能效也伴随较低工作频率和更严格的有效累加范围。
5. **离线算法选择**：论文总体流程中，算法基于图像特征在 off-chip 选择。它还没有给出一个完整的在线场景识别和 L0/L1/L2 动态调度器。

## 6. 横向比较：真正的创新演进

| 维度 | MC-CIM（2022） | CV-CIM（2023/2025） | 对研究方案的启发 |
|---|---|---|---|
| 研究对象 | 匹配代价计算 | 代价体构建 | 研究边界必须明确到 cost construction，而非笼统“加速双目” |
| 基础算子 | XOR/XAC | XOR-derived L0/L1/L2 | 用统一原语降低多算法硬件重复 |
| 计算域 | 模拟阵列 + 数字 HSAT | 数字 Pixel CIM + 模拟 Vector CIM + 数字 DMU | 混合域比全模拟或全数字更合理 |
| 可重构性 | 运算模式、位宽和数据映射 | 算法、精度、映射、行列调度 | 自适应不仅是算法选择，也应包括数据流和精度档位 |
| 鲁棒性 | 对称计算单元改善线性 | 相似性稀疏化 + Canary BIST + PVT 测试 | 需要把“正确性监控”视为架构的一部分 |
| 利用率优化 | 2:1 行地址 MUX、映射复用 | 行跳过 + 灵活行地址 + 奇偶列加载/计算重叠 | FPGA 中可转化为 ping-pong BRAM、双缓冲和窗口预测 |
| 主要能效 | 平均 277 TOPS/W | 峰值 1158 TOPS/W | 不能只比较峰值，应统一算法、位宽、工艺和操作定义 |
| 主要短板 | 利用率和模拟量化 | 仍未覆盖完整 SGBM | 后续创新空间在 cost-SGM 协同，而非继续堆代价计算峰值 |

## 7. 与 SGBM 源码和 FPGA 双目方案的接口边界

完整 SGBM 可抽象为：

```text
图像校正
  -> 特征/代价计算
  -> C(x,y,d) 代价体
  -> 多方向路径聚合 Lr(x,y,d)
  -> 聚合代价 S(x,y,d)
  -> WTA 视差选择
  -> 左右一致性、唯一性、斑点过滤、亚像素优化
```

MC-CIM/CV-CIM 主要对应第二步和第三步的前半段，即生成 `C(x,y,d)`。它们没有解决 SGBM 最有特征性的递推：

```text
Lr(p,d) = C(p,d) + min(
  Lr(p-r,d),
  Lr(p-r,d-1) + P1,
  Lr(p-r,d+1) + P1,
  min_k Lr(p-r,k) + P2
) - min_k Lr(p-r,k)
```

因此，若论文题目写“面向 SGBM 的存内计算加速”，必须说明至少两级接口：

1. 存储近计算单元如何生成或压缩代价体；
2. 代价体是否落到外部存储，还是直接以流式方式送入路径聚合引擎。

真正有价值的系统创新可能不是让代价计算再快一倍，而是 **避免完整代价体反复落存和回读**。

## 8. 推荐的论文方案构思

### 8.1 推荐主线：FPGA 可落地方案

建议题目：

> **面向主动双目 SGBM 的相似性感知近存代价体构建与流式协同架构研究**

这个题目既吸收 CIM 的核心思想，又不把 FPGA BRAM 误称为晶体管级 SRAM-CIM。

建议研究内容分为四层：

#### 研究内容 1：主动散斑场景的代价函数自适应

- 构造局部质量指标：纹理强度、梯度、饱和比例、左右亮度一致性、散斑对比度和初始匹配置信度；
- 在 Census/L0、SAD/L1、梯度或相关性代价之间进行区域级选择或加权融合；
- 将高精度复杂代价只分配给遮挡、弱纹理、反光和重复纹理区域。

可验证假设：主动红外散斑提高了弱纹理区域的可匹配性，但不同曝光、距离和材料会改变局部统计特性，因此固定代价函数不是最优。

#### 研究内容 2：相似性感知差分编码与稀疏代价计算

- 以行块或窗口为单位选择参考像素/参考描述子；
- 对相邻像素或 Census bit-string 做差分/XOR 编码；
- 设计 all-zero bypass、低 popcount 早停和低活动度数据通路；
- 分析 block size 对稀疏率、压缩率、硬件开销和视差精度的影响。

这里应把 CV-CIM 的“提高模拟感知裕量”重新解释为 FPGA 可测量的指标：BRAM 读写次数、总线翻转率、有效 bit 数、动态功耗和周期数。

#### 研究内容 3：代价体构建与 SGM 聚合的流式协同

- 用 line buffer、滑动窗口和视差并行阵列复用右图候选数据；
- 采用 ping-pong BRAM/URAM，使下一块数据加载与当前块计算重叠；
- 不生成完整帧级代价体，而是生成条带或 tile 后直接送入可同步执行的路径聚合方向；
- 对不能完全流式的反向路径，研究条带缓存、压缩中间量或两遍扫描。

这是最可能形成系统级创新的部分，因为它把 CV-CIM 的列自由度从“阵列内部加载/计算重叠”提升到“cost-SGM 两阶段的存储协同”。

#### 研究内容 4：精度、资源和功耗联合调度

- 根据局部置信度动态选择代价位宽、视差搜索范围、窗口大小和路径数；
- 设置保守早停条件，同时保留 worst-case 回退路径；
- 设计在线统计模块，记录跳过率、候选数、代价分布和聚合置信度；
- 用功耗、吞吐和误差的 Pareto 前沿评价，而不是只给峰值 FPS。

### 8.2 进阶主线：面向 ASIC 的真正 SRAM-CIM

若研究条件允许定制芯片、存储编译器和流片，可进一步形成：

> **面向主动双目视觉的可重构 SRAM 存内代价计算与 SGM 近存聚合架构**

相对 CV-CIM，潜在创新点应放在：

- 从 cost-volume construction 扩展到部分 SGM 路径递推；
- 用数字 CIM 或全数字 bitline compute 降低 ADC/PVT 风险；
- 将主动散斑质量作为代价函数和电压/精度调度条件；
- 通过片上互连让代价计算结果直接进入路径聚合，不落外部 DRAM；
- 建立端到端 D1/EPE、能耗/帧、延迟和芯片面积的联合基准。

这条路线技术壁垒更高，但需要明确的 EDA、SRAM 宏、版图和流片条件，不宜仅凭 FPGA 原型宣称实现了 SRAM-CIM。

### 8.3 不建议的方案表述

- “基于存内计算的完整 SGBM 加速器”：若只实现代价计算，范围表述过大。
- “在 FPGA BRAM 中实现模拟 CIM”：技术概念不准确。
- “能效达到 TOPS/W 因而优于现有 FPGA”：不同 operation、工艺和功耗边界不可直接比较。
- “Census、SAD、CNN 三种算法完全自适应”：如果没有在线决策器、切换开销和端到端精度实验，只能称为可配置支持。

## 9. 建议的技术路线图

```text
阶段 1：软件基线
  OpenCV SGBM / 自研 Census-SGM
  -> 分离 cost、aggregation、WTA、post-processing 时间与带宽
  -> 建立 KITTI/Middlebury/自采主动散斑数据集指标

阶段 2：CIM-inspired 代价引擎
  统一 XOR-derived 数据通路
  -> Census/SAD/相关性可配置
  -> 差分稀疏 + all-zero skip
  -> 视差并行和 bit-plane 映射

阶段 3：FPGA 流式协同
  双缓冲加载/计算重叠
  -> tile/条带级代价输出
  -> 与多方向 SGM 聚合引擎直连
  -> 降低完整 cost volume 的落存与回读

阶段 4：场景自适应
  局部图像质量与置信度评估
  -> 动态代价、位宽、视差范围、路径数
  -> 早停与保守回退

阶段 5：ASIC 可行性评估（可选）
  FPGA 活动率/稀疏率统计
  -> 数字 CIM / 混合域 SRAM 宏映射
  -> ADC/PVT/面积/功耗模型
```

## 10. 实验设计和评价指标

### 10.1 数据集与场景

- KITTI 2012/2015：室外驾驶和公开对比；
- Middlebury：室内、高精度和复杂遮挡；
- 自采主动双目：弱纹理、远近距离、黑色/高反/透明材质、重复纹理、强弱环境光、不同散斑曝光；
- 若目标为实时相机，应固定至少两档分辨率与视差范围，例如 720p-D64/D128 和 1080p-D128。

### 10.2 精度指标

- D1-all / bad pixel rate；
- EPE 或 MAE；
- 有效深度比例、空洞率；
- 边缘保持误差；
- 遮挡区、弱纹理区和反光区的分区指标；
- 主动相机可增加绝对深度误差、相对误差和距离分段误差。

### 10.3 硬件和系统指标

- 代价构建延迟与完整 SGBM 延迟；
- FPS、像素吞吐率、视差候选吞吐率；
- 外部内存带宽、BRAM/URAM 读写次数；
- LUT、FF、BRAM、URAM、DSP 和最高频率；
- 板级功耗、能耗/帧、能耗/像素；
- 总线翻转率、稀疏率、跳过率和流水线利用率；
- 代价体未压缩/压缩/不落存三种模式的对比。

### 10.4 必做消融实验

1. 固定代价 vs 自适应代价；
2. 原始编码 vs 相似性差分编码；
3. 无跳过 vs all-zero skip vs 置信度早停；
4. 顺序加载/计算 vs ping-pong 重叠；
5. 完整代价体落存 vs tile 直连 SGM；
6. 固定位宽/固定视差范围 vs 动态精度和动态搜索范围；
7. 不同 block size 对稀疏率、精度和硬件开销的影响。

## 11. 可凝练成论文创新点的表述

以下表述可作为初稿，但必须由实验结果支撑：

1. **提出面向主动双目图像局部质量的多代价自适应机制**，根据散斑可见性、纹理与匹配置信度动态选择 Census、SAD 或相关性代价，在复杂材料与光照条件下平衡精度和计算量。
2. **提出相似性感知差分编码和稀疏代价计算方法**，利用相邻像素连续性降低有效数据活动，并通过零值旁路和置信度早停减少无效视差候选计算。
3. **提出代价体构建与 SGM 路径聚合的流式近存协同架构**，以 tile/条带为调度粒度实现数据加载、代价计算和路径聚合重叠，降低完整代价体的外部存储访问。
4. **提出精度-资源-功耗联合调度策略**，动态配置代价位宽、视差范围和聚合路径数，形成面向实时主动深度相机的可解释性能-精度 Pareto 优化。

其中第 3 点最能与 MC-CIM/CV-CIM 拉开差异：前两代工作主要优化代价计算和代价体构建，而将代价体与 SGM 聚合跨阶段协同，可形成更完整的系统贡献。

## 12. 阅读后仍需进一步回答的问题

1. 当前目标 FPGA 平台的 BRAM/URAM 端口、片上容量和外部 DDR 带宽是多少？
2. 现有 SGBM 源码实际采用哪种初始代价：BT、SAD、Census，还是组合代价？
3. 代价体当前是否完整落 DDR，还是边生成边聚合？
4. 目标是 720p/1080p、30/60 FPS，以及 D64/D128/D256 中的哪一档？
5. 主动双目图像是否包含红外强度、散斑质量或曝光元数据，可否用于在线代价选择？
6. 研究成果边界是 FPGA 原型、ASIC 架构仿真，还是具备流片条件？

这些答案决定最终题目应该使用“近存计算”“存储中心数据流”“CIM-inspired”，还是真正的“SRAM 存内计算”。
