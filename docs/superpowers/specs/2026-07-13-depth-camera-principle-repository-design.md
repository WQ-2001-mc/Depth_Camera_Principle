# 深度相机原理资料库整理设计

## 1. 目标

将现有“深度相机原理总结”目录整理为一个可独立阅读、可追溯来源、可持续维护的中文资料仓库，并发布到 `WQ-2001-mc/Depth_Camera_Principle.git`。

交付结果包括：

- 完善后的 `深度相机原理介绍.md`；
- 一幅可在 GitHub Markdown 中渲染的 Mermaid 分类思维导图；
- 按技术路线整理的代表性 PDF 原理资料；
- 记录资料元数据、来源和收录理由的索引；
- 清晰的仓库入口 `README.md`；
- 独立 Git 历史和经过核验的 GitHub `main` 分支。

## 2. 范围与分类边界

核心资料按五条物理测距路线组织，每类收录 1–2 篇：

1. 被动双目；
2. 单目结构光；
3. 主动双目；
4. dToF；
5. iToF。

dToF 与 iToF 统一放入 `04_ToF`，并分别建立子目录。单目深度学习、多目视觉、扫描 LiDAR、事件相机和多传感器融合会进入综合分类思维导图，帮助说明完整技术版图，但不作为本次批量下载资料的独立类别。

## 4. 资料选择与 Zotero 处理

资料来源分为本机 Zotero 与公开网络两部分。

### 4.1 本机 Zotero

- 从实际 Zotero 数据目录读取数据库和附件，采用只读模式，不修改文献库、集合或标签；
- 以 `ToF`、`time of flight`、`depth camera`、`stereo vision`、`stereo matching`、`structured light`、`active stereo`、`dToF`、`iToF` 等关键词检索标题、摘要、标签、集合名和可提取全文；
- 优先复用已有全文附件；复制进入仓库前记录 Zotero 条目标题和原始来源；
- 用 SHA-256 和 DOI/标题规范化进行去重，避免与联网资料重复；
- Zotero 附件仅在具有开放获取、作者公开稿、厂商公开资料或其他明确可再分发依据时复制；版权状态不明确的文档只在索引中列出来源，不提交 PDF。

### 4.2 公开网络

- 优先顺序为：开放获取经典论文或综述、标准/研究机构资料、厂商官方原理白皮书或数据手册；
- 每类最多 2 篇，选择能够解释原理、又能代表工程实现或典型误差问题的资料；
- 不使用来源不明的转载、网盘扫描件或动态失效页面；
- 下载后验证 PDF 文件头、页数、标题与来源页面，并在索引中记录永久链接或官方链接。

### 4.3 索引字段

`references/资料索引.md` 对每篇文档记录：分类、标题、作者/机构、年份、文档类型、文件路径、DOI 或官方页面、获取来源、收录理由、开放/再分发状态。

## 5. 主文档改造

保留现有“原理—算法—系统瓶颈”的工程导向，并完成以下扩展：

- 明确深度、距离、视差、径向距离和点云坐标的区别；
- 增加深度获取技术全景分类思维导图，区分物理测距与数据驱动推断；
- 为被动双目、单目结构光、主动双目、dToF、iToF 分别给出光路、观测量、核心公式、传统算法、学习算法、主要误差与系统瓶颈；
- 补充共性环节：内外参标定、同步与曝光、深度到点云、RGB-D 配准、置信度和无效值；
- 补充性能评价：精度、重复性、分辨率、量程、完整率、环境光鲁棒性、运动鲁棒性、功耗与数据率；
- 扩展横向对比和应用选型，避免只按标称精度判断；
- 将正文关键事实关联到本地代表性资料或权威在线来源。

思维导图使用 GitHub 支持的 Mermaid `mindmap` 语法；同时在正文提供层级式文字概括，以便不支持 Mermaid 的渲染器仍能理解分类。

## 6. 仓库入口与可维护性

`README.md` 负责说明仓库用途、核心分类、阅读路径、目录结构、资料使用边界和主文档入口。ToF 目录的 `README.md` 解释 dToF 与 iToF 的共同点、差异以及子目录导航。

文件名以稳定、可读为原则：资料 PDF 使用 `年份_第一作者或机构_简短英文标题.pdf`；图片使用英文语义名；中文说明文档保留中文文件名。

## 7. 验证与发布

发布前执行以下检查：

1. 所有 Markdown 本地链接和图片路径均存在；
2. Mermaid 代码块闭合，节点层级和括号不破坏语法；
3. 每个 PDF 能被识别、页数大于零，并与索引标题一致；
4. 五类资料数量各为 1–2 篇，dToF/iToF 均位于 `04_ToF`；
5. 仓库中没有 `.DS_Store`、临时下载或 Zotero 内部数据库文件；
6. Git 差异只包含目标目录内容；
7. 将目标目录作为独立仓库推送到指定远端 `main`，并通过远端引用和网页文件树核验结果。

若 HTTPS Git 连接失败，将在验证现有 GitHub SSH 身份后切换为 SSH 远端。不会把父仓库或兄弟目录的内容纳入提交。

## 8. 完成标准

任务完成时，GitHub 仓库应能直接从 `README.md` 进入完整主文档；分类思维导图可读；每条核心技术路线都有 1–2 篇经筛选、可追溯且允许收录的代表资料；正文无已知断链；本地提交与远端 `main` 指向同一最终提交。
