# 深度相机原理资料库实施计划

**目标：** 按批准的五条测距路线筛选并落盘代表性资料，重构仓库目录，完善中文主文档和 Mermaid 分类思维导图，最终发布并核验 GitHub `main`。

**约束：** Zotero 全程只读；每类 1–2 篇；dToF 与 iToF 位于同一个 ToF 父目录；只提交允许公开分发的 PDF；目标目录作为独立仓库。

## 任务 1：盘点 Zotero 与候选全文

1. 从 Zotero profile 的 `prefs.js` 确认真实数据目录。
2. 使用 SQLite immutable/read-only 模式查询集合、条目、DOI、标签与附件。
3. 以 ToF、深度相机、双目、结构光和主动双目关键词筛选候选。
4. 读取候选 PDF 元数据和必要的首页/摘要文本，输出标题、附件路径与命中依据。

## 任务 2：补充公开资料并确定最终清单

1. 按被动双目、单目结构光、主动双目、dToF、iToF 分别检索 T1 来源。
2. 优先选择开放获取经典论文、研究机构公开稿和厂商官方原理资料。
3. 以 DOI、规范化标题与 SHA-256 对 Zotero 和网络候选去重。
4. 每类保留 1–2 篇，并记录开放许可或官方公开状态。

## 任务 3：建立目标目录并复制资料

1. 创建四个一级技术目录和 ToF 的两个子目录。
2. 将筛选后的 Zotero 附件或下载 PDF 放入对应目录。
3. 验证 PDF 文件头、页数、可提取标题与文件名。
4. 建立 `references/资料索引.md` 和 `docs/04_ToF/README.md`。

## 任务 4：整理图片与仓库入口

1. 将现有图片迁移至 `assets/images` 并改为语义化文件名。
2. 修复主文档全部图片路径和替代文本。
3. 删除已被新结构替代的空目录、Finder 元数据和临时文件。
4. 新建仓库根 `README.md` 与 `.gitignore`。

## 任务 5：完善主文档与分类思维导图

1. 增加术语、坐标和测量量定义。
2. 增加完整 Mermaid `mindmap`，区分物理测距、数据驱动推断与融合路线。
3. 补齐共性成像链路、标定、同步、误差、置信度、点云和 RGB-D 配准。
4. 校订五类核心路线的原理、公式、算法、瓶颈与代表产品。
5. 扩充性能指标、横向比较、应用选型和参考资料导航。

## 任务 6：自动与人工验证

1. 检查 Markdown 本地链接、图片路径和代码块闭合。
2. 检查每类 PDF 数量、PDF 页数、索引条目和 SHA-256。
3. 渲染或解析 PDF 代表页，确认文件有效且文字/图示可读。
4. 执行 `git diff --check`，检查仓库无临时文件和越界内容。

## 任务 7：发布 GitHub

1. 检查 `gh` 与 GitHub 身份，并核对目标远端状态。
2. 仅暂存目标仓库内容并创建清晰提交。
3. 推送 `main` 到 `WQ-2001-mc/Depth_Camera_Principle.git`；必要时使用 SSH。
4. 通过 `git ls-remote` 和 GitHub 文件树核对远端最终提交。
